Sort Algorithms

Types of sort algorithms

Sort algorithms can be classified by:

  • The place where the sort takes place:
    • Internal sort algorithms: in the main computer memory.
    • External sort algorithms: in an external place like a secondary memory.
  • The time the algorithm takes to compute the sort, with the input given already sorted or inversely sorted:
    • Natural sort algorithms: it takes the minimum possible time when the input is ordered.
    • Non natural sort algorithms: it takes the minimum possible time when the input is inversely ordered.
  • The steadiness of the algorithm:
    • An stable sort keeps the relative order that the elements with same keys had originally. For example: if a list sort by dates is resorted in alphabetical order with an stable sort algorithm, all of the elements with the same alphabetical key will be sorted by date. A non stable sort algorithm can be implemented to be a stable sort algorithm. One way to do that would be to modify the order key, so the relative original position on the list can participate on the sort in case of coincidence.

Characteristics

  • Computational complexity (best, worst and average case complexity) in terms of size n. The best sort behavior (unless you don't use the key structure) is $${O(n log n)}$$. The most simple algorithms are quadratic, $${O(n^2)}$$.
  • Memory usage and other computational means. Also $${O(n)}$$ notation is used.
